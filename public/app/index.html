動画プロジェクト

台本・画像を用意するだけで動画が完成します

① 台本（video-config.json）

Gemini などで出した JSON を貼り付けて保存。各シーンに text・effect・durationInSeconds を含めてください。

② 画像アップロード

ファイル名の先頭に 01_ 02_ … を付けると順番が固定されます。アップロード後「マニフェスト更新」を押してください。

直接コピーしても OK → public/drop/images/

③-① 音声自動生成

TTS プロバイダーを選択して API キーを入力し、音声を生成してください。

英語の2人会話: speech_text に A: [excited] Hello! / B: [curious] Really? のように行を分けるか、 dialogue 配列を使います。シーンごとに話者だけ変える場合は "speaker": "a" または "b"。 Dialogue API

0.5x（ゆっくり）1.0x（普通）1.5x（速い）

③-② 読み上げ修正

テロップ（表示の text）はそのままで、読み上げだけ変えられます。読み上げ欄が空欄のシーンは、台本 JSON の speech_text と同じ内容が読まれます（テロップと同じではありません）。

台本を保存すると自動で一覧が表示されます。

③-③ 読み上げ辞書（全体置換ルール）

よく間違えられる単語や漢字を一括で修正します。保存すると、全シーンの読み上げ用テキストに自動適用されます。

置換前（間違える単語） 置換後（正しいひらがな）

④ テロップスタイル

動画に重なるテロップの見た目を設定します。保存するとすぐプレビューに反映されます。

⑤ エクスポート

音声生成・画像アップ・マニフェスト更新が済んだらエクスポートできます。レンダーには数分〜数十分かかります。進捗ログは npm run app を実行しているターミナル（PowerShell など）に表示されます。ブラウザは完了まで待機表示のままになります。

⑥ テロップ直接編集

テキストを変更すると自動保存され、プレビューにリアルタイム反映されます。

台本を保存すると自動で一覧が表示されます。

⑦ プレビュー（Remotion Studio）

「起動」を押すと Remotion Studio がここに表示されます

⑧ 新しい動画を始める

前の動画の台本 JSON・生成音声・画像・書き出し（out/export）・テロップスタイル・読み上げ辞書をすべて削除し、初期状態に戻します。元に戻せません。

消しません: .env、API キー用の環境変数、node_modules、src などのソースコード。

ボタンが失敗するとき: プロジェクトフォルダでターミナルを開き npm run reset-project はい を実行すると、ブラウザなしで同じ消去ができます。

前作データをすべて消去しますか？

次のものが完全に削除または初期化されます。

  • video-config.json（台本）→ 空 []
  • telop-style.json（テロップ見た目）→ 初期値
  • dictionary.json（読み上げ辞書）→ 空
  • assets-manifest.json → 音声・画像 0 件
  • public/drop/audio 内の全ファイル
  • public/drop/images 内の全ファイル
  • out/export 内のすべて（過去の MP4 / SRT など）
  • out 直下の動画・音声ファイル（例: Studio が出した *.mp4）

実行するには、下の欄に はい または はい！ とだけ入力してください（コピペ可）。
